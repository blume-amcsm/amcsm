AMCSM – MANIFEST
Advanced Musical Communication & Semantic Mapping
Full Version (English)
1. Preamble

AMCSM is a private research project exploring a new paradigm:

    A model can speak music.

Music is not treated as an audio file but as a semantic language, communicated through MIDI‑2.0, emotion, and real‑time interaction.
This manifesto outlines the conceptual foundations, goals, and philosophical orientation of the project.

It is not a technical specification but a foundational document that defines the direction of the work.
2. Fundamental Assumptions
2.1 Music is a language

Music exhibits structures that functionally resemble linguistic systems:

    syntax

    semantics

    pragmatics

    prosody

    contextual meaning

Music is a system of meaning, intention, emotion, and interaction.
A model should not predict notes but communicate musical meaning.

2.2 MIDI‑2.0 is a semantic format

MIDI‑2.0 provides high‑resolution parameters that are not merely technical values but carriers of expression:

    timing nuances

    dynamic contours

    articulation

    pitch modulation

    per‑note controllers

    property exchange

These parameters form a semantic space in which musical meaning can be encoded precisely.

2.3 Emotion is a continuous control space

Emotion is not treated as a label but as a continuous parameter space (e.g., valence/arousal).
This space governs both musical expression and prosodic speech behavior.

2.4 Music is interaction, not output

Music emerges through dialogue between agents.
A model should not “generate something” but respond, adapt, co‑regulate, and mean.

3. Paradigm Shift
3.1 From generator to communicator

AMCSM is not:

    an audio generator

    a diffusion model

    a note predictor

    a Magenta‑style system

AMCSM is a communication system for exchanging musical meaning.

3.2 Meaning over data volume

Music is not understood as a large dataset but as a compressed unit of meaning.
A single MIDI‑2.0 event can carry more semantic information than thousands of audio tokens.

3.3 Expression over style

Style is a byproduct.
The primary focus is on:

    intention

    emotion

    timing

    articulation

    interaction

4. Transfer to Voice

A central idea of AMCSM:

    The mechanisms that communicate musical meaning can be applied directly to voice.

Musical parameters correspond structurally to prosodic parameters:

    timing ↔ speech rate

    dynamics ↔ loudness contour

    articulation ↔ consonant/vowel shaping

    pitch nuances ↔ intonation

    emotion ↔ prosodic modulation

This makes AMCSM a universal expression model, treating music and speech as two dialects of the same semantic structure.

5. Technical Guidelines
5.1 Tokenization

MIDI‑2.0 events are treated as semantic tokens.
A token represents an intention, not merely a parameter.

5.2 Real‑time capability

The system must be:

    low‑latency

    deterministic

    bidirectional

    adaptive

to support musical interaction.

5.3 Property Exchange

Models should be able to explicitly describe their capabilities, including:

    resolution

    expressive range

    emotion‑mapping

    timing precision

    interaction modes

This enables transparency and adaptive communication.

6. Research Goals

    Define a musical semantic framework

    Map emotion to musical parameters

    Develop a real‑time interaction model

    Transfer the mechanisms to voice and prosody

    Evaluate the system with musicians across genres

    Publish all foundational ideas as open prior art

    Establish a universal model of expressive communication

7. Ethics & Openness

AMCSM is:

    non‑commercial

    independent of institutions

    open to scientific collaboration

    transparent in its intentions

    defensively published (prior art)

The project aims to preserve freedom for creative and scientific work and prevent the privatization of foundational ideas.

8. Closing Statement

AMCSM is an attempt to understand music and voice as a shared expressive language.
A model should not imitate but communicate.
Not generate, but mean.
Not replace, but interact.

This manifesto is the starting point.
The technical implementation will evolve step by step.
